# KO Copy Guide

## 목표

설정/알림/오류 문구를 한국어 사용자 기준으로 일관되고 즉시 이해 가능한 형태로 유지한다.

## 스타일 원칙

- 짧고 직접적으로 작성한다.
- 기능명은 UI 라벨과 동일한 용어를 사용한다.
- 에러 문구는 `원인 + 바로 할 일` 순서로 제시한다.
- 한국어/영어 혼합 시 한국어를 먼저 두고, 영문은 괄호 또는 보조 설명으로 처리한다.

## 금지 패턴

- 모호한 표현: "문제가 발생했습니다"만 단독 사용
- 원인 없는 실패 문구
- 같은 기능에 서로 다른 번역어 혼용

## 권장 패턴

- 필수값 누락: `필수값 누락: OAuth client ID를 입력하세요.`
- URI 불일치: `Redirect URI가 일치하지 않습니다. 콘솔에 http://127.0.0.1:8765/callback 을 등록하세요.`
- 외부 의존성 누락: `soffice를 찾을 수 없습니다. LibreOffice 설치 후 경로를 확인하세요.`

## 핵심 용어집

- Quick Start: 빠른 시작
- Login (OAuth): 로그인(OAuth)
- Models & Provider: 모델/제공자
- Parser: 파서
- Advanced: 고급
- Chat model profile: 채팅 모델 프로필
- OAuth session actions: OAuth 세션 동작
- Parser ingest output format: 파서 출력 형식

## 번역 리뷰 체크

- 같은 라벨이 다른 탭/문서에서 동일 번역인지 확인
- 명령형 버튼은 동사 중심으로 통일 (`검증`, `적용`, `로그인 시작`)
- 보안 관련 문구는 경고 의도를 분명히 표현 (`위험`, `주의`, `권장하지 않음`)
